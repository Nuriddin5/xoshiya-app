## Manual UI Check

1. Run `npm run dev`.
2. Open the Electron window.
3. Go to Settings.
4. Enter a valid local Whisper executable path in `Whisper binary path`.
5. Enter a valid local model file path in `Whisper model path`.
6. Save settings and confirm no validation error appears.
7. Go to Dashboard.
8. Select a desktop source that provides audio.
9. Start recording and let one chunk complete.
10. Confirm a new chunk first shows `saving`, then `transcribing`, then `saved`.
11. Confirm transcript text appears inside the chunk record.
12. Confirm the local chunk file path is shown under the transcript.
13. Stop recording and confirm the recorder status becomes `stopped`.
14. Repeat with an invalid Whisper binary path and confirm the failed chunk shows a useful launch or missing-file error.
15. Repeat with an invalid Whisper model path and confirm the failed chunk shows a useful model-path error.

## Failures Fixed In Place

- Review finding: the runner used the short `-otxt` flag, which has a known upstream failure mode in some `whisper.cpp` versions where the text file is not written.
- Fix: changed the command to use `--model`, `--file`, `--language`, `--output-txt`, and `--output-file`.
- Review finding: the runner had no process timeout and could hang indefinitely on a stuck local CLI invocation.
- Fix: added a five-minute timeout that kills the child process and returns a clear timeout error.
