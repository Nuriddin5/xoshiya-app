# Manual UI Verification Report - Rubai Worker Speed Optimization

**Task:** Make Rubai transcription significantly faster.
**Date:** 2026-05-13

## Investigation & Profiling
The previous implementation of the `rubai_worker.py` script read requests from `stdin` sequentially and called the `transcribe` function one by one. This forced the entire transcription pipeline into a strict serial execution, negating any potential multicore performance benefits on Windows. The `faster-whisper` backend was properly kept alive across chunks, but the renderer and IPC queue were also heavily serializing the requests with `createSerialTaskQueue()`.

## Changes Made
1. **Worker Concurrency**: Refactored `scripts/rubai_worker.py` to use `concurrent.futures.ThreadPoolExecutor` with a default concurrency of 2. `WhisperModel` was instantiated with `num_workers=2`, making it thread-safe and capable of concurrent transcription processing.
2. **Task Queue Update**: Updated `src/shared/task-queue.ts` from a serial queue to a generic `createTaskQueue(concurrency: number)`. Both `src/renderer/audio-recorder.ts` and `src/main/ipc.ts` were updated to allow a queue concurrency of 2.
3. **Chunk Duration**: Changed `DEFAULT_CHUNK_SECONDS` from 30 to 15 seconds. Processing smaller chunks concurrently ensures lower latency, higher system resource utilization, and faster presentation to the user.

## Automated Verification
- `npm run typecheck`: Passed
- `npm test`: Passed (with `task-queue.test.js` completely rewritten to assert concurrent behavior)
- `npm run build`: Passed

## Performance Results (Estimated)
- **Before**: 30-second chunks processed strictly one-by-one. High latency before the first text appeared.
- **After**: 15-second chunks processed with a concurrency of 2. Text appears twice as fast initially, and total processing throughput is significantly improved without reloading the model or converting audio redundantly.

## Manual Verification Steps
1. Launch the app and go to the Dashboard.
2. Ensure the Rubai status shows "ASR ready".
3. Start recording from a desktop source playing a video or podcast.
4. Observe the chunks appearing every 15 seconds (instead of 30 seconds).
5. Verify that multiple chunks move from `pending` to `transcribing` and `done` without halting the recording or locking up the app.
6. Check that the final merged transcript remains complete and continuous.
